# xion-types Usage Guide

## What This Package Provides

`@burnt-labs/xion-types` is a **fully generated** TypeScript type library for XION blockchain. It contains:

1. **CosmWasm contract types** ŌĆö generated by `@cosmwasm/ts-codegen` from contract JSON schemas
   - `contracts/account/` ŌĆö AbstractAccount contract messages and queries
   - `contracts/treasury/` ŌĆö Treasury contract messages and queries
   - `contracts/*/schema/raw/*.json` ŌĆö Raw CosmWasm JSON Schema files (included in dist)

2. **Proto types** ŌĆö generated by `buf` + `ts-proto` from BSR (Buf Schema Registry)
   - `abstractaccount/v1/` ŌĆö XION-specific AbstractAccount proto types
   - `cosmos/`, `cosmwasm/`, `ibc/`, `google/` ŌĆö Standard Cosmos SDK proto types

> **Do not add manual code to this package.** All content is regenerated on each xion release.

## int64 / Long vs bigint

### Current State

The ts-proto-generated proto types currently use `Long` (from the `long` library) for int64/uint64 fields. `@cosmjs/*` 0.32.x and `cosmjs-types@0.9.0` use native `bigint`. This creates a type incompatibility at cosmjs interface boundaries.

### The Fix (pending next release)

`buf/buf.gen.ts.yaml` now sets `forceLong=bigint`:

```yaml
opt: "esModuleInterop=true,forceLong=bigint,useOptionals=messages,outputClientImpl=grpc-web"
```

After the next xion release (which triggers CI codegen), all proto types will use `bigint` and be fully compatible with `cosmjs-types@0.9.0`.

### Running Codegen Manually

```bash
cd xion-types

# Regenerate TypeScript types from BSR (requires buf CLI + internet, no Go needed)
XION_BSR_MODULE="buf.build/burnt-labs/xion:v26.0.0" ./scripts/proto-gen-ext.sh ts

# Then build the dist
cd ts && pnpm run build
```

## How to Use in xion.js

### Use xion-types for XION-specific types

```typescript
// Xion-specific proto types ŌĆö no cosmjs-types equivalent
import { AbstractAccount } from '@burnt-labs/xion-types/abstractaccount/v1/account';
import { MsgRegisterAccount } from '@burnt-labs/xion-types/abstractaccount/v1/tx';

// CosmWasm contract types ŌĆö from ts-codegen, not ts-proto
import type { FeeConfig, GrantConfig, Params, TreasuryAddr } from '@burnt-labs/xion-types';
```

### Use cosmjs-types for standard cosmos types (current workaround)

Until xion-types is regenerated with `forceLong=bigint`, use `cosmjs-types` for any type that sits at a cosmjs interface boundary (types with int64/uint64 fields):

```typescript
import { SignDoc, TxRaw, AuthInfo, Fee } from 'cosmjs-types/cosmos/tx/v1beta1/tx';
import { SignMode } from 'cosmjs-types/cosmos/tx/signing/v1beta1/signing';
import { ServiceClientImpl, SimulateRequest } from 'cosmjs-types/cosmos/tx/v1beta1/service';
import { MsgGrant, MsgExec } from 'cosmjs-types/cosmos/authz/v1beta1/tx';
import { GenericAuthorization } from 'cosmjs-types/cosmos/authz/v1beta1/authz';
import { ContractExecutionAuthorization, MaxCallsLimit } from 'cosmjs-types/cosmwasm/wasm/v1/authz';
```

Once xion-types is regenerated with bigint, consolidate all imports to `@burnt-labs/xion-types`.

## Contract JSON Schema Files

The dist includes raw CosmWasm JSON Schema files. Load them at runtime for OpenAPI schema generation:

```typescript
// In argus ŌĆö load schema for OpenAPI docs
const treasurySchema = require('@burnt-labs/xion-types/contracts/treasury/schema/raw/instantiate.json');
```
